2026-05-29 강의자료 03: 리스트 - 모아모아 다모아

리스트는 여러 값을 하나로 묶어 저장하는 자료형이다. 학생 이름, 점수 목록, 상품 목록처럼 비슷한 성격의 값을 함께 다룰 때 유용하다. 리스트 만들기 scores = [90, 80, 70] names = ["사용자 A", "사용자 B", "사용자 C"] 리스트는 대괄호를 사용하고, 값들은 쉼표로 구분한다. 인덱스 리스트의 위치 번호를 인덱스라고 한다. 파이썬에서 첫 번째 위치는 1이 아니라 0이다. scores = [90, 80, 70] print(scores[0]) print(scores[1]) scores[0]은 첫 번째 값인 90을 의미한다. 반복과 활용 리스트는 반복문과 함께 쓰면 더 강력하다. names = ["사용자 A", "사용자 B", "사용자 C"] for name in names: print(name) 오늘 가져갈 것 리스트는 데이터를 여러 개 다루기 위한 기본 도구다. 데이터 분석에서도 표의 열이나 행을 다룰 때 여러 값을 묶어 처리하는 사고가 필요하다.

2026-05-28 강의자료 04: 보안

보안은 서비스를 만든 뒤 추가로 붙이는 장식이 아니다. 로그인, 개인정보, 권한, 데이터 저장을 다루는 순간부터 함께 생각해야 한다. 해시 해시는 데이터를 일정한 규칙으로 바꾼 값이다. 비밀번호는 원문 그대로 저장하면 위험하기 때문에 보통 해시를 사용한다. 중요한 점은 원래 값을 쉽게 되돌릴 수 없게 관리해야 한다는 것이다. 쿠키와 세션 HTTP는 기본적으로 이전 요청을 기억하지 않는다. 로그인 상태를 유지하려면 쿠키나 세션 같은 방법이 필요하다. 쿠키는 브라우저에 저장되는 작은 정보이고, 세션은 서버가 사용자 상태를 기억하는 방식이다.

2026-05-28 강의자료 02: 인터넷과 HTTP

HTTP는 웹에서 클라이언트와 서버가 대화하는 규칙이다. 브라우저가 서버에 요청을 보내고, 서버가 상태 코드와 데이터를 응답으로 돌려준다. HTTP 요청과 응답 사용자가 웹페이지에 접속하면 브라우저는 서버에 요청을 보낸다. 서버는 요청을 처리한 뒤 HTML, CSS, 이미지, JSON 같은 데이터를 돌려준다. 요청에는 주소, 메서드, 헤더, 필요한 데이터가 포함될 수 있다. 응답에는 상태 코드와 실제 데이터가 들어온다. 상태 코드 코드 의미 200 성공 301/302 이동 400 잘못된 요청 401 인증 필요 403 권한 없음 404 찾을 수 없음 500 서버 오류 오류 코드를 읽을 수 있으면 문제 위치를 빠르게 좁힐 수 있다.

2026-05-28 강의자료 01: 네트워크 개요

네트워크는 컴퓨터들이 서로 데이터를 주고받을 수 있게 연결된 구조다. 웹사이트 접속, 메신저, 이메일, 파일 다운로드는 모두 네트워크 위에서 움직인다. IP 주소 IP 주소는 네트워크에서 장치를 찾기 위한 주소다. 집 주소가 있어야 택배가 도착하듯이, 컴퓨터도 IP 주소가 있어야 데이터를 주고받을 수 있다. 도메인 이름은 사람이 읽기 쉽게 만든 이름이고, 실제 통신에서는 DNS를 통해 IP 주소로 바뀐다. 통신 방식 네트워크 통신은 데이터를 작은 단위로 나누어 보내고, 상대방이 다시 조립하는 방식으로 이해할 수 있다. 데이터가 한 번에 통째로 이동하는 것이 아니라 여러 조각이 규칙에 따라 움직인다.

2026-05-27 강의자료 05: 리눅스의 세계로

리눅스는 서버 환경에서 많이 쓰이는 운영체제다. 개인 컴퓨터에서는 윈도우를 많이 쓰지만, 웹서비스를 배포하고 운영하는 서버에서는 리눅스를 자주 만난다. 리눅스를 배우는 이유 리눅스는 안정적이고 비용 부담이 낮으며, 서버 자동화 도구와 잘 연결된다. 클라우드 서버를 만들면 기본 환경이 리눅스인 경우도 많다. 개발자가 로그를 확인하거나, 파일을 수정하거나, 서버 프로세스를 재시작할 때 리눅스 명령어를 사용한다. CLI 환경 CLI는 글자로 명령을 입력하는 환경이다. 처음에는 버튼이 없어서 어렵게 느껴지지만, 반복 작업을 자동화하기 좋다. 예를 들어 ls로 파일 목록을 보고, cd로 디렉터리를 이동하고, mkdir로 폴더를 만든다. 이런 명령어가 익숙해지면 서버에서 직접 문제를 확인할 수 있다.

2026-05-27 강의자료 04: 파일 시스템

파일 시스템은 컴퓨터가 파일과 디렉터리를 정리하는 방식이다. 사용자는 폴더와 파일을 눈으로 보지만, 운영체제는 정해진 규칙에 따라 저장 위치와 이름을 관리한다. 파일과 디렉터리 파일은 데이터를 담는 단위이고, 디렉터리는 파일을 묶어두는 공간이다. 디렉터리 안에 또 다른 디렉터리가 들어갈 수 있어서 트리 구조가 만들어진다. 경로는 파일의 위치를 나타낸다. 윈도우에서는 C:\Users\... 같은 형태를 많이 쓰고, 리눅스에서는 /home/user/... 같은 형태를 많이 쓴다. 개발에서 중요한 이유 개발할 때는 파일 위치가 자주 문제를 만든다. 실행 위치가 다르면 상대 경로가 달라지고, 파일 권한이 없으면 읽거나 쓸 수 없다.

2026-05-27 강의자료 03: 메모리

메모리는 프로그램이 실행되는 동안 데이터를 올려두는 작업 공간이다. 저장 장치가 책장이라면, 메모리는 지금 펼쳐 놓고 쓰는 책상에 가깝다. 메모리가 하는 일 프로그램을 실행하면 운영체제는 필요한 코드와 데이터를 메모리에 올린다. CPU는 메모리에 올라온 데이터를 빠르게 읽고 계산한다. 메모리가 부족하면 컴퓨터는 느려진다. 책상 위 공간이 부족하면 책을 계속 꺼냈다 넣었다 해야 하는 것처럼, 컴퓨터도 데이터를 옮기는 데 시간이 더 든다. 개발과 메모리 개발자는 변수를 만들고 리스트에 값을 넣고 객체를 생성한다. 이런 값들은 실행 중에 메모리를 사용한다. 데이터가 많아지는 프로그램에서는 메모리 사용량을 생각해야 한다.

2026-05-27 강의자료 02: 프로세스

프로세스는 실행 중인 프로그램이다. 프로그램은 저장 장치에 있는 파일이고, 그 프로그램이 실제로 실행되어 메모리에 올라오면 프로세스가 된다. 프로그램과 프로세스 메모장 프로그램 파일이 있다고 해서 항상 실행 중인 것은 아니다. 사용자가 메모장을 열면 운영체제가 실행 공간을 만들고, 그때부터 프로세스가 된다. 프로세스는 코드, 데이터, 실행 상태, 메모리 공간을 가진다. 운영체제는 여러 프로세스가 동시에 움직일 수 있도록 스케줄링한다. 왜 중요한가 서버에서 웹 애플리케이션을 실행하면 그것도 하나의 프로세스다. 서비스가 멈췄을 때 프로세스가 살아 있는지, CPU나 메모리를 너무 많이 쓰고 있는지 확인해야 한다.

2026-05-27 강의자료 01: 운영체제 소개

운영체제는 컴퓨터를 사용할 수 있게 해주는 기본 소프트웨어다. 사용자는 프로그램을 실행한다고 느끼지만, 실제로는 운영체제가 CPU, 메모리, 파일, 입출력 장치를 관리한다. 운영체제의 역할 운영체제는 컴퓨터의 관리자다. 여러 프로그램이 동시에 실행될 때 어떤 프로그램에 CPU 시간을 줄지, 메모리를 얼마나 배정할지, 파일을 어디에 저장할지 조율한다. 예를 들어 브라우저, 메신저, 코드 편집기를 동시에 켜도 컴퓨터가 어느 정도 질서 있게 움직이는 이유는 운영체제가 자원을 나눠주기 때문이다. 사용자와 하드웨어 사이 사용자는 하드웨어를 직접 제어하지 않는다. 파일을 저장하거나 프로그램을 실행하는 행동은 운영체제를 통해 하드웨어에 전달된다.

2026-05-26 강의자료 02: 앱·웹 서비스를 위한 모든 것

이 자료의 핵심은 서비스가 화면만으로 완성되지 않는다는 점이다. 사용자가 보는 버튼과 페이지 뒤에는 서버, 데이터베이스, 관리자 기능, 보안, 개인정보 처리 흐름이 함께 있다. 핵심 정리 웹서비스는 클라이언트와 서버가 요청과 응답을 주고받는 구조다. 클라이언트는 사용자가 보는 브라우저나 앱이고, 서버는 뒤에서 실제 처리를 담당하는 컴퓨터다. 사용자가 버튼을 누르면 클라이언트가 서버에 요청을 보낸다. 서버는 로그인 여부를 확인하거나, 데이터를 저장하거나, 비즈니스 규칙을 적용한 뒤 응답을 돌려준다. 서비스 기획에서 봐야 할 것 초보자는 사용자 화면만 생각하기 쉽다. 하지만 실제 서비스에서는 관리자 화면도 필요하다. 예를 들어 명함 촬영 서비스를 만든다면 사용자는 사진을 업로드하고, 관리자는 결과를 검수하거나 데이터를 수정할 수 있어야 한다.

2026-05-26 강의자료 01: 컴퓨터의 이해

이 자료는 IT 리터러시의 출발점에 해당한다. 핵심은 컴퓨터가 사람의 말을 바로 이해하는 것이 아니라, 사람이 작성한 코드를 컴퓨터가 처리할 수 있는 형태로 바꾸어 실행한다는 점이다. 핵심 정리 컴퓨터는 계산과 저장을 빠르게 처리하는 기계다. 사람은 파이썬이나 자바스크립트 같은 프로그래밍 언어로 명령을 작성하고, 인터프리터나 컴파일러가 그 명령을 컴퓨터가 이해할 수 있는 형태로 바꿔준다. 흐름은 단순하게 보면 다음과 같다. 사람이 코드를 작성한다. 인터프리터나 컴파일러가 번역한다. 컴퓨터가 명령을 실행한다. 결과가 화면, 파일, 네트워크 응답 등으로 나타난다. 인터프리터는 코드를 비교적 바로 읽으면서 실행하는 방식이고, 파이썬이 대표적이다. 컴파일러는 실행 전에 코드를 한 번에 번역하는 방식이고, C나 C++에서 자주 만난다.
