오늘 수업에서 데이터 분석 파트는 코딩보다 “설명"에 가까웠다. 표를 만들고 그래프를 그리는 건 분석의 끝이 아니라 중간 과정이다. 진짜 중요한 건 그 숫자가 무슨 뜻인지 말할 수 있는가다. 데이터 분석이란 무엇인가 데이터 분석은 데이터를 보고 의미를 찾아내고, 그 의미를 다른 사람이 이해할 수 있게 설명하는 일이다. 방문자 수 그래프를 만들었다고 분석이 끝나는 건 아니다. 방문자가 왜 줄었는지, 어느 페이지에서 많이 나갔는지, 다음에는 무엇을 바꾸면 좋을지를 말할 수 있어야 한다.

2026-05-26 이어드림스쿨 수업 정리: REST API와 데이터베이스 기초

데이터베이스와 REST API 자료는 오늘 내용 중 가장 서비스 개발에 가까운 파트였다. 웹서비스가 화면과 서버의 대화라면, API는 그 대화의 약속이고 데이터베이스는 기억을 저장하는 공간이다. 처음엔 용어가 많아서 복잡해 보이지만, 큰 흐름은 이렇다. 클라이언트가 API로 요청한다 → 서버가 비즈니스 로직을 처리한다 → 서버가 DB에 쿼리한다 → 결과를 응답한다 API와 REST API API는 프로그램끼리 데이터를 주고받는 약속된 통로다. 사람이 메뉴판을 보고 주문하듯이, 프로그램도 API라는 약속을 보고 요청한다. REST API는 웹에서 많이 쓰는 API 방식이다. 주소와 요청 방식으로 데이터를 다룬다.

2026-05-26 이어드림스쿨 수업 정리: 앱·웹 서비스는 화면 뒤에서 어떻게 돌아갈까

앱_웹 서비스를 위한 모든 것 자료에서 핵심은 하나였다. 서비스는 예쁜 화면만으로 완성되지 않는다. 사용자가 보는 화면 뒤에는 서버, 데이터베이스, 관리자 기능, 보안, 개인정보 처리까지 줄줄이 붙어 있다. 초보자는 화면을 먼저 본다. 버튼이 있고, 입력창이 있고, 결과가 보이면 서비스처럼 느껴진다. 하지만 실제 운영 가능한 서비스는 버튼을 누른 뒤에 무슨 일이 일어나는지가 더 중요하다. 클라이언트와 서버 클라이언트는 사용자가 직접 보는 쪽이다. 웹브라우저, 스마트폰 앱, 화면에 보이는 버튼과 입력창이 여기에 가깝다.

2026-05-26 이어드림스쿨 수업 정리: 컴퓨터는 코드를 어떻게 이해할까

오늘 컴퓨터의 이해 자료에서 가장 크게 잡은 감각은 이것이다. 사람은 사람이 읽기 쉬운 말로 코드를 쓰고, 컴퓨터는 그 코드를 바로 읽는 게 아니라 자기 방식으로 번역해서 실행한다. 처음에는 “파이썬 코드를 치면 컴퓨터가 그대로 알아듣는 것 아닌가?“라고 생각하기 쉽다. 그런데 실제로는 중간에 번역자가 있다. 그 번역자가 인터프리터이거나 컴파일러다. 코드가 실행되는 흐름 흐름만 보면 생각보다 단순하다. 사람이 코드를 작성한다. 인터프리터나 컴파일러가 코드를 번역한다. 컴퓨터가 이해할 수 있는 형태로 바뀐다. 컴퓨터가 명령을 실행한다. 인터프리터는 코드를 한 줄씩 읽고 실행하는 방식이다. 파이썬이 대표적이다. 선생님이 문장을 한 줄씩 읽어주면 학생이 바로 따라 하는 느낌에 가깝다.
