2026-05-27 강의자료 03: 메모리

메모리는 프로그램이 실행되는 동안 데이터를 올려두는 작업 공간이다. 저장 장치가 책장이라면, 메모리는 지금 펼쳐 놓고 쓰는 책상에 가깝다.

메모리가 하는 일

프로그램을 실행하면 운영체제는 필요한 코드와 데이터를 메모리에 올린다. CPU는 메모리에 올라온 데이터를 빠르게 읽고 계산한다.

메모리가 부족하면 컴퓨터는 느려진다. 책상 위 공간이 부족하면 책을 계속 꺼냈다 넣었다 해야 하는 것처럼, 컴퓨터도 데이터를 옮기는 데 시간이 더 든다.

개발과 메모리

개발자는 변수를 만들고 리스트에 값을 넣고 객체를 생성한다. 이런 값들은 실행 중에 메모리를 사용한다. 데이터가 많아지는 프로그램에서는 메모리 사용량을 생각해야 한다.

서버에서도 메모리는 중요하다. 방문자가 많거나 데이터 처리가 무거우면 메모리가 부족해져 서비스가 느려질 수 있다.

오늘 가져갈 것

메모리는 눈에 잘 보이지 않지만 프로그램 성능과 안정성에 직접 연결된다. 코드가 실행될 때 어디에 올라가고, 어떤 데이터를 들고 있는지 상상할 수 있으면 디버깅에 도움이 된다.
