2026-05-27 강의자료 03: 메모리

이 글은 [수업자료] 메모리.pdf를 보고 다시 정리한 복습 기록이다. 슬라이드를 그대로 옮기기보다는, 나중에 봤을 때 “아, 이게 그 말이었지” 하고 떠올릴 수 있게 쉬운 말과 예시 중심으로 적었다.

한 줄 요약

메모리는 프로그램이 실행되는 동안 필요한 데이터를 올려두는 작업 공간이고, 운영체제는 이 공간을 나누고 보호한다.

수업 흐름

컴퓨터에는 저장장치와 메모리가 따로 있다. 저장장치는 오래 보관하는 창고이고, 메모리는 지금 당장 작업하는 책상이다. 프로그램을 실행하면 필요한 코드와 데이터가 메모리에 올라간다. 메모리가 부족하면 운영체제는 가상 메모리를 이용해 디스크 일부를 보조 공간처럼 쓰기도 하지만, 속도는 느려질 수 있다.

쉽게 이해하기

책상 비유가 가장 편하다. 책장이 저장장치라면, 책상은 메모리다. 책상이 넓으면 여러 책과 노트를 펼쳐 놓고 빠르게 작업할 수 있다. 책상이 좁으면 필요한 자료를 계속 책장에서 꺼냈다 넣었다 해야 해서 느려진다.

헷갈렸던 단어

용어 내가 이해한 뜻
RAM 실행 중인 프로그램과 데이터를 올려두는 빠른 임시 저장 공간
가상 메모리 메모리가 부족할 때 디스크 일부를 보조 공간처럼 쓰는 방식
스택 함수 호출과 지역 변수처럼 순서가 중요한 데이터를 담는 영역
힙 실행 중 필요에 따라 동적으로 할당되는 데이터 영역

짧은 예시

데이터 분석에서 큰 엑셀 파일을 여러 개 열면 메모리를 많이 쓴다. RAM이 부족하면 프로그램이 느려지거나 멈출 수 있다. 그래서 대용량 데이터를 다룰 때는 필요한 열만 읽거나, 데이터를 나눠 처리하는 습관이 필요하다.

복습 체크

  • 메모리와 저장장치의 차이를 책상과 책장 비유로 설명할 수 있는가?
  • 가상 메모리가 편하지만 느려질 수 있다는 점을 이해했는가?
  • 프로그램 실행 중 메모리 사용량을 확인해야 하는 상황을 떠올릴 수 있는가?

오늘의 정리

메모리는 눈에 보이지 않지만 성능 체감에 바로 영향을 준다. 앞으로 프로그램이 느릴 때 ‘코드가 문제인가, 데이터가 너무 큰가, 메모리가 부족한가’를 나눠 생각해야겠다.
