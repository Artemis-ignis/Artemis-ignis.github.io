2026-05-28 강의자료 02: 인터넷과 HTTP

이 글은 [수업 자료] 인터넷과 HTTP.pdf를 보고 다시 정리한 복습 기록이다. 슬라이드를 그대로 옮기기보다는, 나중에 봤을 때 “아, 이게 그 말이었지” 하고 떠올릴 수 있게 쉬운 말과 예시 중심으로 적었다.

한 줄 요약

HTTP는 웹브라우저와 서버가 요청과 응답으로 대화하기 위한 대표적인 약속이다.

수업 흐름

웹은 요청과 응답의 반복으로 움직인다. 사용자가 링크를 누르거나 버튼을 누르면 브라우저가 서버에 HTTP 요청을 보내고, 서버는 HTML, JSON, 이미지 같은 응답을 돌려준다. 상태 코드는 요청 결과를 숫자로 알려 준다. 200번대는 성공, 300번대는 이동, 400번대는 클라이언트 쪽 문제, 500번대는 서버 쪽 문제로 크게 볼 수 있다.

쉽게 이해하기

HTTP는 손님과 직원의 대화 같다. 손님이 ‘이 메뉴 주세요’라고 요청하면 직원은 ‘여기 있습니다’ 또는 ‘재고가 없습니다’라고 답한다. 요청이 명확해야 응답도 제대로 오고, 문제가 생기면 상태 코드가 힌트를 준다.

헷갈렸던 단어

용어 내가 이해한 뜻
URL 웹에서 자원의 위치를 나타내는 주소
HTTP 메서드 GET, POST처럼 요청의 행동을 나타내는 방식
상태 코드 요청 결과를 숫자로 표현한 약속
캐시 자주 쓰는 데이터를 잠시 저장해 빠르게 다시 쓰는 방식

짧은 예시

로그인 페이지를 열 때는 GET 요청으로 화면을 가져오고, 아이디와 비밀번호를 제출할 때는 POST 요청을 보낸다. 서버가 성공을 응답하면 로그인 상태를 유지하기 위해 쿠키나 세션을 사용할 수 있다.

복습 체크

  • HTTP가 요청과 응답 구조라는 점을 설명할 수 있는가?
  • 200, 404, 500 같은 상태 코드의 느낌을 구분할 수 있는가?
  • 쿠키와 캐시가 사용자 경험과 성능에 영향을 준다는 점을 이해했는가?

오늘의 정리

HTTP는 API와 웹페이지를 이해하는 공통 언어다. 앞으로 개발자 도구 Network 탭을 볼 때 요청 URL, 메서드, 상태 코드부터 확인하면 흐름이 보일 것 같다.
