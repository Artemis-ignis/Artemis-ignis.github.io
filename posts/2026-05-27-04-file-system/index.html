2026-05-27 강의자료 04: 파일 시스템

이 글은 [수업자료] 파일 시스템.pdf를 보고 다시 정리한 복습 기록이다. 슬라이드를 그대로 옮기기보다는, 나중에 봤을 때 “아, 이게 그 말이었지” 하고 떠올릴 수 있게 쉬운 말과 예시 중심으로 적었다.

한 줄 요약

파일 시스템은 데이터를 파일과 폴더 구조로 정리하고, 어디에 무엇이 있는지 운영체제가 찾을 수 있게 만드는 방식이다.

수업 흐름

우리가 보는 파일 이름 뒤에는 경로, 확장자, 크기, 생성 시간, 수정 시간, 권한 같은 정보가 붙어 있다. 운영체제는 이 정보를 이용해 파일을 찾고 읽고 쓴다. 파일 시스템이 없다면 저장장치 안의 데이터는 그냥 긴 숫자 덩어리처럼 보일 것이다.

쉽게 이해하기

파일 시스템은 큰 도서관의 분류 체계와 비슷하다. 책 제목만 있는 게 아니라 어느 층, 어느 서가, 어떤 번호에 있는지 정리되어 있어야 빠르게 찾을 수 있다. 폴더는 서가, 파일은 책, 경로는 책을 찾는 주소라고 생각하면 된다.

헷갈렸던 단어

용어 내가 이해한 뜻
파일 운영체제가 다루는 데이터 저장 단위
디렉터리 파일과 다른 디렉터리를 담는 폴더 구조
절대 경로 최상위 위치부터 시작하는 전체 주소
권한 누가 읽고 쓰고 실행할 수 있는지 정한 규칙

짧은 예시

Windows에서는 C:\Users\사용자\Documents처럼 경로를 쓰고, 리눅스에서는 /home/user/documents처럼 쓴다. 모양은 다르지만 둘 다 파일이 어디 있는지 알려 주는 주소다.

복습 체크

  • 파일과 디렉터리의 차이를 설명할 수 있는가?
  • 절대 경로와 상대 경로를 구분할 수 있는가?
  • 파일 권한이 보안과 운영에 연결된다는 점을 이해했는가?

오늘의 정리

파일 시스템은 개발을 시작하면 매일 마주치는 기본기다. 경로를 잘못 써서 파일을 못 찾는 오류가 많기 때문에, 지금부터 절대 경로와 상대 경로를 차분히 구분하는 습관이 필요하다.
