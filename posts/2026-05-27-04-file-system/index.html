2026-05-27 강의자료 04: 파일 시스템

파일 시스템은 컴퓨터가 파일과 디렉터리를 정리하는 방식이다. 사용자는 폴더와 파일을 눈으로 보지만, 운영체제는 정해진 규칙에 따라 저장 위치와 이름을 관리한다.

파일과 디렉터리

파일은 데이터를 담는 단위이고, 디렉터리는 파일을 묶어두는 공간이다. 디렉터리 안에 또 다른 디렉터리가 들어갈 수 있어서 트리 구조가 만들어진다.

경로는 파일의 위치를 나타낸다. 윈도우에서는 C:\Users\... 같은 형태를 많이 쓰고, 리눅스에서는 /home/user/... 같은 형태를 많이 쓴다.

개발에서 중요한 이유

개발할 때는 파일 위치가 자주 문제를 만든다. 실행 위치가 다르면 상대 경로가 달라지고, 파일 권한이 없으면 읽거나 쓸 수 없다.

Hugo 블로그에서도 content/posts에 있는 마크다운 파일과 public/posts에 생성되는 HTML 파일은 역할이 다르다. 위치를 정확히 알아야 배포 흐름을 이해할 수 있다.

오늘 가져갈 것

파일 시스템은 개발자의 기본 지도다. 어디에 원본이 있고, 어디에 결과물이 생기며, 어떤 경로를 기준으로 명령이 실행되는지 확인하는 습관이 중요하다.
