2026-05-27 강의자료 05: 리눅스의 세계로

이 글은 [강의자료] 리눅스의 세계로.pdf를 보고 다시 정리한 복습 기록이다. 슬라이드를 그대로 옮기기보다는, 나중에 봤을 때 “아, 이게 그 말이었지” 하고 떠올릴 수 있게 쉬운 말과 예시 중심으로 적었다.

한 줄 요약

리눅스는 서버와 개발 자동화에서 많이 쓰이는 운영체제라서, 웹서비스를 배포하고 운영하려면 기본 사용법을 익혀야 한다.

수업 흐름

개인 컴퓨터에서는 Windows를 많이 쓰지만, 실제 서버 환경은 리눅스 기반인 경우가 많다. 비용이 낮고, 안정적으로 오래 켜 두기 좋고, 자동화 도구와 배포 도구가 리눅스 중심으로 돌아가는 경우가 많기 때문이다. Windows 사용자도 WSL Ubuntu를 쓰면 리눅스 환경을 연습할 수 있다.

쉽게 이해하기

리눅스는 개발자가 서버 방에 들어갈 때 쓰는 공용 언어처럼 느껴졌다. 내 노트북에서는 버튼으로 하던 일을 서버에서는 명령어로 처리한다. 처음에는 불친절해 보이지만, 익숙해지면 반복 작업을 빠르게 자동화할 수 있다.

헷갈렸던 단어

용어 내가 이해한 뜻
배포 내가 만든 서비스를 사용자가 접근할 수 있는 서버에 올리는 일
터미널 명령어로 컴퓨터와 대화하는 화면
쉘 명령어를 해석해서 운영체제에 전달하는 프로그램
WSL Windows에서 리눅스 환경을 사용할 수 있게 해 주는 기능

짧은 예시

Hugo 블로그도 로컬에서는 파일을 작성하고, 빌드하고, GitHub에 푸시해서 공개된다. 이런 작업 흐름은 나중에 웹서비스를 서버에 배포할 때도 비슷하게 이어진다.

복습 체크

  • 서버에서 리눅스가 많이 쓰이는 이유를 세 가지 이상 말할 수 있는가?
  • 터미널과 쉘의 관계를 대략 이해했는가?
  • Windows 사용자도 WSL로 리눅스 명령어를 연습할 수 있다는 점을 알고 있는가?

오늘의 정리

리눅스는 처음부터 깊게 파기보다, 파일 이동, 명령 실행, 로그 확인, 권한 수정처럼 자주 쓰는 작업부터 몸에 익히는 게 좋겠다. 서버 운영의 첫 문턱이라고 생각하면 된다.
