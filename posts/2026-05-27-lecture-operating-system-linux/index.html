운영체제는 관리자다

운영체제는 컴퓨터 자원을 관리하는 기본 프로그램이다. 사용자가 프로그램을 실행하면 운영체제가 CPU, 메모리, 파일, 입출력 장치를 조율한다.

쉽게 말하면 운영체제는 학교의 교무실 같은 역할을 한다. 어떤 수업이 어느 교실을 쓰는지, 누가 먼저 처리되어야 하는지, 자료는 어디에 보관할지 관리한다.

프로세스와 메모리

프로세스는 실행 중인 프로그램이다. 파일로 저장되어 있는 프로그램이 실제로 실행되면 운영체제는 그 프로그램에 작업 공간을 배정한다. 이 작업 공간에는 코드, 데이터, 임시 계산값, 실행 흐름이 들어간다.

메모리는 프로그램이 실행되는 동안 사용하는 작업대다. 저장 장치가 책장이라면, 메모리는 지금 펼쳐 놓고 보는 책상이다. 책상 공간이 부족하면 작업이 느려지거나 정리 비용이 커진다.

파일 시스템과 경로

파일 시스템은 파일과 폴더를 정리하는 규칙이다. 윈도우에서는 C:\Users\... 같은 경로를 많이 보고, 리눅스에서는 /home/user/... 같은 경로를 많이 본다.

리눅스 파일 시스템은 /를 기준으로 뻗어 나간다. /home은 사용자 파일, /etc는 설정 파일, /var는 로그나 변하는 데이터, /bin과 /usr/bin은 실행 파일과 관련이 깊다.

리눅스를 배우는 이유

웹서비스를 배포하고 운영할 때 리눅스 서버를 많이 쓴다. 비용이 낮고, 서버 환경에서 안정적으로 쓰이고, 배포 자동화 도구도 리눅스를 기준으로 설명되는 경우가 많다.

윈도우에서 개발하더라도 WSL Ubuntu 같은 리눅스 환경에 익숙해지면 좋다. 실제 서버에서 파일을 찾고, 로그를 보고, 프로세스를 확인하고, 권한 문제를 해결할 때 리눅스 명령어가 필요하다.

자주 쓰는 명령어 감각

명령어 의미
pwd 현재 위치 확인
ls 파일 목록 보기
cd 폴더 이동
mkdir 폴더 만들기
rm 파일 삭제
cp 복사
mv 이동 또는 이름 변경
cat 파일 내용 출력
grep 문자열 검색

CLI는 처음에는 불친절해 보이지만 반복 작업에 강하다. 버튼을 하나씩 누르는 대신 명령어를 기록해두면 같은 작업을 다시 실행하거나 자동화하기 쉽다.

오늘의 정리

운영체제와 리눅스는 개발자가 서버를 다루기 위한 기본 체력이다. 웹서비스를 만들 때 코드만 아는 것으로는 부족하다. 프로그램이 어떤 프로세스로 실행되는지, 메모리와 파일은 어떻게 관리되는지, 서버에서는 어떤 명령어로 상태를 확인하는지 알아야 문제를 직접 추적할 수 있다.
