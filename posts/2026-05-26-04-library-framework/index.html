2026-05-26 강의자료 04: 라이브러리와 프레임워크

이 자료는 개발자가 왜 남이 만든 도구를 쓰는지 설명한다. 모든 기능을 처음부터 직접 만들 수도 있지만, 실제 개발에서는 이미 검증된 라이브러리와 프레임워크를 활용하는 일이 많다.

라이브러리

라이브러리는 필요할 때 가져다 쓰는 도구 모음이다. 내가 주도권을 가지고 필요한 기능만 호출한다.

pandas는 파이썬에서 표 데이터를 다루는 대표적인 라이브러리다. 엑셀이나 CSV 파일을 읽고, 필요한 열을 고르고, 그룹별로 집계하고, 결측치를 처리할 때 자주 사용한다.

프레임워크

프레임워크는 기본 구조를 제공하는 틀이다. 개발자는 그 틀 안에서 약속된 방식으로 코드를 작성한다.

Django는 파이썬으로 웹사이트와 백엔드를 만들 때 쓰는 웹 프레임워크다. URL 연결, 데이터베이스 모델, 관리자 페이지, 인증 같은 기능을 체계적으로 구성할 수 있다.

차이점

라이브러리가 공구함이라면 프레임워크는 작업장에 가깝다. 라이브러리는 내가 필요할 때 꺼내 쓰고, 프레임워크는 정해진 구조 안에서 개발 흐름을 따라간다.

오늘 가져갈 것

도구를 잘 쓰는 것도 개발 실력이다. pandas는 데이터 분석에 가깝고, Django는 웹서비스 제작에 가깝다. 둘의 목적과 사용 장면을 구분해서 기억하면 좋다.
