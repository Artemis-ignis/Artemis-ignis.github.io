2026-05-26 강의자료 04: 라이브러리와 프레임워크

이 글은 [수업자료] 라이브러리와 프레임워크.pdf를 보고 다시 정리한 복습 기록이다. 슬라이드를 그대로 옮기기보다는, 나중에 봤을 때 “아, 이게 그 말이었지” 하고 떠올릴 수 있게 쉬운 말과 예시 중심으로 적었다.

한 줄 요약

라이브러리는 내가 필요할 때 꺼내 쓰는 도구이고, 프레임워크는 내가 그 안에 들어가 규칙을 따라 만드는 작업 틀이다.

수업 흐름

개발을 하다 보면 모든 기능을 직접 만들 수 없다. 그래서 남들이 잘 만들어 둔 도구를 가져다 쓴다. pandas처럼 특정 작업을 도와주는 것은 라이브러리라고 부르기 쉽고, Django처럼 프로젝트 구조와 실행 흐름까지 잡아 주는 것은 프레임워크라고 이해하면 된다. 둘 다 생산성을 높여 주지만, 내가 주도권을 얼마나 갖는지가 다르다.

쉽게 이해하기

요리로 비유하면 라이브러리는 칼, 도마, 믹서기 같은 도구다. 내가 요리 순서를 정하고 필요할 때 도구를 꺼낸다. 프레임워크는 밀키트나 조리 매뉴얼이 있는 주방에 가깝다. 정해진 순서와 구조를 따르면 빠르게 결과를 만들 수 있지만, 그 규칙을 이해해야 편하다.

헷갈렸던 단어

용어 내가 이해한 뜻
라이브러리 필요한 기능을 가져다 쓰는 도구 모음
프레임워크 프로젝트 구조와 흐름을 제공하는 개발 틀
pandas 표 형태 데이터를 읽고 정리하고 분석하는 파이썬 라이브러리
Django 파이썬으로 웹사이트와 백엔드를 만들 때 쓰는 웹 프레임워크

짧은 예시

엑셀 파일을 분석할 때 pd.read_excel()을 쓰면 pandas 라이브러리의 기능을 빌리는 것이다. 반대로 회원가입, 로그인, 관리자 페이지가 있는 웹서비스를 만들 때 Django 프로젝트 구조를 따라가면 프레임워크 안에서 개발하는 것이다.

복습 체크

  • 라이브러리와 프레임워크를 ‘주도권’ 기준으로 설명할 수 있는가?
  • pandas가 데이터 분석 도구이고 Django가 웹서비스 제작 도구라는 차이를 말할 수 있는가?
  • 새 도구를 배울 때 공식 문서와 예제를 같이 보는 습관을 떠올릴 수 있는가?

오늘의 정리

이번 자료를 보고 도구 이름을 외우는 것보다 역할을 구분하는 게 먼저라는 생각이 들었다. 앞으로 새 기술을 만나면 ‘내가 불러 쓰는 도구인가, 내가 들어가서 규칙을 따라야 하는 틀인가’부터 확인해야겠다.
