2026-05-26 강의자료 02: 앱·웹 서비스를 위한 모든 것

이 자료의 핵심은 서비스가 화면만으로 완성되지 않는다는 점이다. 사용자가 보는 버튼과 페이지 뒤에는 서버, 데이터베이스, 관리자 기능, 보안, 개인정보 처리 흐름이 함께 있다.

핵심 정리

웹서비스는 클라이언트와 서버가 요청과 응답을 주고받는 구조다. 클라이언트는 사용자가 보는 브라우저나 앱이고, 서버는 뒤에서 실제 처리를 담당하는 컴퓨터다.

사용자가 버튼을 누르면 클라이언트가 서버에 요청을 보낸다. 서버는 로그인 여부를 확인하거나, 데이터를 저장하거나, 비즈니스 규칙을 적용한 뒤 응답을 돌려준다.

서비스 기획에서 봐야 할 것

초보자는 사용자 화면만 생각하기 쉽다. 하지만 실제 서비스에서는 관리자 화면도 필요하다. 예를 들어 명함 촬영 서비스를 만든다면 사용자는 사진을 업로드하고, 관리자는 결과를 검수하거나 데이터를 수정할 수 있어야 한다.

명함에는 이름, 회사, 전화번호, 이메일 같은 개인정보가 들어갈 수 있다. 그래서 수집 동의, 접근 권한, 삭제 요청, 보관 기간도 같이 생각해야 한다.

오늘 가져갈 것

좋은 서비스 기획은 예쁜 화면을 그리는 데서 끝나지 않는다. 사용자의 행동, 서버 처리, 데이터 저장, 관리자 운영, 보안과 법적 책임까지 이어서 보는 것이 실제 서비스에 가까운 사고방식이다.
