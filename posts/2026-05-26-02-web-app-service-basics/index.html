2026-05-26 강의자료 02: 앱·웹 서비스를 위한 모든 것

이 글은 [수업자료] 앱_웹 서비스를 위한 모든 것.pdf를 보고 다시 정리한 복습 기록이다. 슬라이드를 그대로 옮기기보다는, 나중에 봤을 때 “아, 이게 그 말이었지” 하고 떠올릴 수 있게 쉬운 말과 예시 중심으로 적었다.

한 줄 요약

웹서비스는 예쁜 화면 하나가 아니라, 사용자 화면과 서버 처리, 데이터 저장, 관리자 운영, 보안이 같이 움직이는 시스템이다.

수업 흐름

서비스를 기획할 때 초보자는 보통 ‘사용자가 보는 화면’부터 떠올린다. 하지만 실제 서비스는 사용자가 버튼을 누른 뒤 서버에서 무엇을 확인하고, 어떤 데이터를 저장하고, 관리자는 어디서 검수하고, 문제가 생기면 어떻게 복구할지까지 생각해야 한다. 화면은 입구이고, 뒤쪽에는 백엔드 로직과 데이터베이스, 인증, 권한, 로그, 분석이 붙는다.

쉽게 이해하기

식당으로 생각하면 이해가 쉽다. 손님이 보는 메뉴판은 프론트엔드, 주문을 받아 주방으로 넘기는 과정은 API, 실제 요리하는 주방은 백엔드, 재료 창고와 주문 기록 장부는 데이터베이스다. 식당을 제대로 운영하려면 메뉴판만 예쁘게 만들면 안 되고, 재고 관리와 결제, 위생, 직원용 화면까지 필요하다.

헷갈렸던 단어

용어 내가 이해한 뜻
프론트엔드 사용자가 직접 보는 화면과 상호작용 부분
백엔드 로그인, 저장, 계산, 조회처럼 뒤에서 처리되는 부분
관리자 페이지 운영자가 데이터를 확인하고 수정하는 내부용 화면
개인정보 이름, 연락처, 이메일처럼 개인을 알아볼 수 있는 정보

짧은 예시

명함 촬영 서비스를 만든다고 하면 사용자는 사진을 찍어 올리는 화면만 본다. 하지만 서버는 이미지를 받고, 텍스트를 추출하고, DB에 저장하고, 관리자는 잘못 인식된 정보를 수정할 수 있어야 한다. 명함에는 이름과 전화번호가 있으니 수집 동의, 보관 기간, 삭제 요청도 같이 생각해야 한다.

복습 체크

  • 서비스 기획에서 사용자 화면과 관리자 화면을 분리해서 생각할 수 있는가?
  • 개인정보가 들어가는 기능에서 동의, 권한, 삭제 요청을 떠올릴 수 있는가?
  • 프론트엔드와 백엔드가 어떤 기준으로 나뉘는지 설명할 수 있는가?

오늘의 정리

이 자료는 서비스가 화면으로만 끝나지 않는다는 걸 보여준다. 앞으로 아이디어를 떠올릴 때는 ‘사용자는 무엇을 누르나’와 함께 ‘서버는 무엇을 처리하나’, ‘데이터는 어디에 남나’, ‘운영자는 어디서 관리하나’를 같이 적어야겠다.
