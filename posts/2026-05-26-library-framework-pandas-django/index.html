2026-05-26 이어드림스쿨 수업 정리: 라이브러리와 프레임워크, pandas와 Django

라이브러리와 프레임워크 자료를 보면서 가장 먼저 정리한 건 이 차이다. 라이브러리는 필요한 도구를 꺼내 쓰는 느낌이고, 프레임워크는 집을 지을 때 기본 골조가 이미 잡혀 있는 느낌이다.

둘 다 남이 미리 만들어둔 도움을 쓰는 것이지만, 주도권이 조금 다르다.

라이브러리는 공구함에 가깝다

라이브러리는 남들이 미리 만들어둔 도구 모음이다. 내가 필요할 때 가져다 쓴다.

예를 들어 엑셀 파일을 직접 한 줄씩 해석하려면 힘들다. 그런데 pandas를 쓰면 표 데이터를 쉽게 불러오고 다룰 수 있다.

import pandas as pd

df = pd.read_excel("data.xlsx")

pd.read_excel() 한 줄로 엑셀 파일을 불러올 수 있다는 점이 pandas의 강점이다. 물론 실제 프로젝트에서는 파일 경로, 시트 이름, 결측치, 데이터 타입도 확인해야 한다. 그래도 처음부터 모든 기능을 직접 만들 필요가 없다는 점이 크다.

pandas는 데이터 분석 도구다

pandas는 파이썬에서 표 데이터를 다루는 대표적인 라이브러리다. 엑셀처럼 행과 열로 된 데이터를 읽고, 정리하고, 집계하고, 분석할 때 자주 쓴다.

주로 이런 일을 한다.

  1. 엑셀이나 CSV 파일 읽기
  2. 필요한 열만 선택하기
  3. 결측치 확인하기
  4. 그룹별 합계나 평균 구하기
  5. 분석하기 좋은 형태로 데이터 정리하기

초보자 입장에서는 pandas를 “파이썬 안의 엑셀 작업 도우미"라고 생각하면 이해가 쉽다.

프레임워크는 기본 구조에 가깝다

프레임워크는 단순한 도구 하나보다 더 큰 틀이다. 웹서비스를 만들 때 필요한 기본 구조를 제공하고, 개발자는 그 구조 안에서 기능을 채운다.

Django는 “장고"라고 읽는다. 파이썬으로 웹사이트와 백엔드를 만들 때 쓰는 웹 프레임워크다.

Django는 이런 기능을 만들 때 도움을 준다.

  1. URL 연결
  2. 화면 처리
  3. 데이터베이스 모델
  4. 로그인과 인증
  5. 관리자 페이지
  6. 서버 쪽 로직 구성

작은 스크립트를 만들 때는 pandas 같은 라이브러리만으로 충분할 수 있다. 하지만 로그인, 게시판, 관리자 페이지, 데이터베이스가 있는 웹서비스를 만들려면 Django 같은 프레임워크가 훨씬 편하다.

라이브러리와 프레임워크의 차이

구분 라이브러리 프레임워크
비유 공구함 건물 골조
주도권 내가 필요할 때 호출 정해진 구조 안에서 작성
예시 pandas, requests Django, Flask, Spring
잘 맞는 일 특정 기능 해결 서비스 전체 구조 만들기

핵심은 “누가 흐름을 잡고 있느냐"다. 라이브러리는 내가 필요할 때 부른다. 프레임워크는 프레임워크가 정한 흐름에 맞춰 내가 코드를 넣는다.

내 식으로 정리

pandas와 Django는 둘 다 파이썬 생태계의 도구지만 목적이 다르다.

pandas는 데이터를 분석하기 위한 도구다. Django는 웹서비스를 만들기 위한 구조다. 같은 파이썬을 쓰더라도 하나는 표 데이터를 정리하는 데 강하고, 하나는 서버와 웹서비스를 구성하는 데 강하다.

오늘 정리한 기준은 앞으로 새 도구를 배울 때도 쓸 수 있을 것 같다. “이건 작은 기능을 도와주는 라이브러리인가, 아니면 전체 구조를 잡아주는 프레임워크인가?“라고 먼저 물어보면 된다.
