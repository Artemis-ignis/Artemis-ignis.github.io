2026-05-26 이어드림스쿨 수업 정리: 데이터 분석은 그래프보다 설명이다

오늘 수업에서 데이터 분석 파트는 코딩보다 “설명"에 가까웠다. 표를 만들고 그래프를 그리는 건 분석의 끝이 아니라 중간 과정이다. 진짜 중요한 건 그 숫자가 무슨 뜻인지 말할 수 있는가다.

데이터 분석이란 무엇인가

데이터 분석은 데이터를 보고 의미를 찾아내고, 그 의미를 다른 사람이 이해할 수 있게 설명하는 일이다.

방문자 수 그래프를 만들었다고 분석이 끝나는 건 아니다. 방문자가 왜 줄었는지, 어느 페이지에서 많이 나갔는지, 다음에는 무엇을 바꾸면 좋을지를 말할 수 있어야 한다.

양적 연구는 숫자와 통계 중심의 분석이다. 방문자 수, 매출, 클릭 수, 체류 시간처럼 숫자로 볼 수 있는 데이터를 다룬다.

질적 연구는 사람과 맥락을 이해하는 분석이다. 인터뷰, 관찰, 후기처럼 숫자만으로는 알기 어려운 이유를 살펴본다.

이어드림스쿨에서는 주로 양적 연구를 배우지만, 좋은 분석가는 숫자 뒤의 맥락도 봐야 한다. 숫자는 현상을 보여주고, 맥락은 이유를 생각하게 해준다.

데이터 분석 흐름

분석 흐름은 이렇게 정리할 수 있다.

  1. 데이터 수집
  2. 데이터 정리
  3. 결측치와 이상치 처리
  4. 그룹별 집계
  5. 시각화
  6. 통계적 해석
  7. 인사이트 도출
  8. 보고서 작성
  9. 발표와 의사결정 연결

처음엔 이 순서가 길어 보이지만, 실제로는 “재료 준비 → 요리 → 맛 설명 → 다음 메뉴 제안” 같은 흐름이다.

결과와 인사이트는 다르다

결과는 데이터로 확인된 사실이다.

방문자 수가 20% 줄었다.

이 문장은 결과다. 숫자로 확인한 사실을 말한다.

인사이트는 숫자의 의미와 다음 행동을 설명한다.

모바일 첫 화면 이탈률이 높아졌기 때문에 버튼 위치와 문구를 개선해야 한다.

이 문장은 인사이트에 가깝다. 단순히 줄었다고 말하는 데서 끝나지 않고, 왜 그런지 추정하고 다음 행동으로 연결한다.

보고서에는 무엇이 들어가야 할까

분석 보고서는 코드 모음이 아니다. 의사결정에 쓸 수 있는 설명이어야 한다.

보고서에는 최소한 이런 내용이 들어가면 좋다.

  1. 분석 목적
  2. 사용 데이터
  3. 핵심 결과
  4. 인사이트
  5. 원인 추정
  6. 개선 제안
  7. 한계와 추가 확인 사항

특히 한계와 추가 확인 사항을 적는 게 중요하다. 확인하지 않은 내용을 확정처럼 쓰면 분석이 아니라 추측이 된다.

Google Analytics와 웹분석 도구

웹이나 앱을 만든 뒤에는 Google Analytics 같은 웹분석 도구를 붙여 사용자 행동을 볼 수 있다.

확인할 수 있는 지표는 예를 들어 이런 것들이다.

지표 볼 수 있는 것
방문자 수 얼마나 많은 사람이 왔는지
페이지 조회수 어떤 페이지가 많이 열렸는지
체류 시간 사용자가 얼마나 머물렀는지
이탈률 어디서 나갔는지
버튼 클릭 어떤 행동을 했는지
전환율 회원가입이나 구매까지 갔는지

예를 들어 회원가입 버튼은 많이 눌리는데 가입 완료율이 낮다면, 가입 과정이 복잡하거나 입력 항목이 많을 수 있다.

분석 도구를 붙이는 것 자체가 목적은 아니다. 분석 도구는 사용자의 행동을 이해하고 서비스를 더 좋게 만들기 위한 눈이다.

내 식으로 정리

데이터 분석의 최종 결과물은 코드가 아니라 설명이다. 코드는 분석을 도와주는 도구이고, 그래프는 설명을 돕는 그림이다.

분석가는 숫자를 보고 “그래서 무엇을 해야 하는가"까지 말할 수 있어야 한다. 오늘 배운 데이터 분석은 결국 서비스를 더 나은 방향으로 바꾸기 위한 대화 도구에 가깝다.
