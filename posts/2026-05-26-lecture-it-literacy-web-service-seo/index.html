컴퓨터와 프로그래밍 언어

사람이 쓴 코드를 컴퓨터가 바로 이해하는 것은 아니다. 파이썬, 자바스크립트 같은 언어는 사람이 이해하기 쉽게 만든 표현이고, 실행할 때 인터프리터나 컴파일러가 컴퓨터가 처리할 수 있는 형태로 바꿔준다.

쉽게 말하면 코드는 사람의 명령서이고, 인터프리터와 컴파일러는 번역가다. 컴퓨터는 최종적으로 번역된 명령을 실행한다.

웹서비스는 화면만이 아니다

웹서비스를 처음 볼 때는 버튼, 입력창, 페이지 같은 화면이 먼저 보인다. 하지만 실제 서비스는 화면 뒤쪽에서 훨씬 많은 일이 일어난다.

사용자가 버튼을 누르면 브라우저나 앱이 서버에 요청을 보낸다. 서버는 로그인 여부를 확인하거나, 데이터베이스에서 정보를 찾거나, 결제나 주문 같은 규칙을 처리한다. 그 결과를 다시 클라이언트에게 응답으로 돌려준다.

식당으로 비유하면 클라이언트는 손님이 보는 주문 화면이고, 서버는 주방과 계산대다. 손님은 메뉴만 보지만, 실제 처리는 뒤쪽에서 일어난다.

API와 데이터베이스

API는 프로그램끼리 데이터를 주고받기 위한 약속된 통로다. REST API에서는 주소와 요청 방식으로 무엇을 할지 표현한다.

요청 방식 의미 예시
GET 조회 글 목록 보기
POST 생성 새 글 작성
PATCH 수정 글 내용 수정
DELETE 삭제 글 삭제

서버는 API 요청을 받은 뒤 데이터베이스에 쿼리를 보낸다. 데이터베이스의 기본 동작은 CRUD로 정리할 수 있다.

CRUD SQL 게시판 예시
Create INSERT 글 작성
Read SELECT 글 조회
Update UPDATE 글 수정
Delete DELETE 글 삭제

JSON과 XML은 데이터를 주고받는 형식이다. 요즘 웹 API에서는 JSON을 많이 쓰지만, 공공데이터나 오래된 시스템에서는 XML도 자주 만난다.

라이브러리와 프레임워크

라이브러리는 필요할 때 꺼내 쓰는 도구 모음이다. pandas는 파이썬에서 표 데이터를 다루는 대표적인 라이브러리다. 엑셀 파일을 읽고, 필요한 열을 고르고, 그룹별 집계를 할 때 유용하다.

프레임워크는 더 큰 틀이다. Django는 파이썬으로 웹서비스와 백엔드를 만들 때 사용하는 프레임워크다. 라이브러리가 공구함이라면, 프레임워크는 기본 구조가 잡혀 있는 작업장에 가깝다.

현업 용어와 SEO

현업에서 자주 나오는 용어도 서비스 개발과 연결된다. 버전 관리는 코드 변경 이력을 남기는 일이고, Git과 GitHub가 대표적이다. 오픈소스는 공개된 코드를 함께 개선하는 방식이다. 클라우드는 서버를 직접 사서 운영하지 않고 필요한 만큼 빌려 쓰는 방식이고, SaaS는 설치 없이 웹에서 바로 쓰는 서비스 형태다.

SEO는 검색엔진 최적화다. 좋은 글을 쓰는 것만으로는 부족하고, 제목, 설명, URL, 문서 구조가 검색엔진이 이해하기 쉽게 정리되어야 한다. Hugo 블로그에서도 title, description, tags, categories 같은 front matter를 잘 쓰는 이유가 여기에 있다.

오늘의 정리

이날 수업은 IT 기초 용어를 외우는 시간이 아니라, 실제 웹서비스가 어떻게 구성되는지 큰 지도를 그리는 시간이었다. 화면, 서버, API, 데이터베이스, 개발 도구, 검색 노출은 따로 떨어진 주제가 아니라 하나의 서비스 안에서 서로 연결된다.
