2026-05-28 강의자료 01: 네트워크 개요

이 글은 [수업 자료] 네트워크 개요.pdf를 보고 다시 정리한 복습 기록이다. 슬라이드를 그대로 옮기기보다는, 나중에 봤을 때 “아, 이게 그 말이었지” 하고 떠올릴 수 있게 쉬운 말과 예시 중심으로 적었다.

한 줄 요약

네트워크는 여러 컴퓨터가 약속된 주소와 규칙을 이용해 데이터를 작은 조각으로 주고받는 구조다.

수업 흐름

인터넷을 쓸 때 우리는 주소창에 도메인을 입력하지만, 실제 통신은 IP 주소와 패킷을 중심으로 움직인다. DNS가 도메인을 IP로 바꿔 주고, 데이터는 작은 패킷으로 나뉘어 여러 장비를 지나 목적지까지 간다. 라우터와 스위치 같은 장비는 데이터를 어디로 보낼지 결정한다.

쉽게 이해하기

택배를 생각하면 쉽다. 도메인은 사람이 읽기 쉬운 이름이고, IP 주소는 실제 배송 주소다. 패킷은 큰 짐을 여러 상자로 나눈 것이고, 라우터는 중간 물류센터처럼 다음 길을 정한다.

헷갈렸던 단어

용어 내가 이해한 뜻
IP 주소 네트워크에서 장비를 찾기 위한 주소
DNS 도메인 이름을 IP 주소로 바꿔 주는 시스템
패킷 네트워크로 보내기 위해 작게 나눈 데이터 조각
라우터 패킷이 다음에 어디로 갈지 길을 정하는 장비

짧은 예시

브라우저에 example.com을 입력하면 먼저 DNS가 IP 주소를 찾는다. 그다음 내 컴퓨터는 서버로 요청 패킷을 보내고, 서버는 응답 패킷을 돌려준다. 우리는 그 결과를 웹페이지로 본다.

복습 체크

  • 도메인과 IP 주소의 차이를 설명할 수 있는가?
  • 패킷이 왜 필요한지 택배 비유로 말할 수 있는가?
  • 라우터와 스위치가 네트워크 장비라는 점을 이해했는가?

오늘의 정리

네트워크는 눈에 보이지 않지만 웹서비스의 길이다. 길이 막히면 서비스도 느려지거나 끊기기 때문에, 개발자도 기본 네트워크 흐름을 알아야 한다.
