2026-05-26 강의자료 03: 데이터베이스와 REST API

이 자료는 웹서비스가 데이터를 어떻게 주고받고 저장하는지 설명한다. 화면에서 버튼을 누르는 일은 단순해 보이지만, 뒤에서는 API 요청과 데이터베이스 처리가 이어진다.

API와 REST API

API는 프로그램끼리 데이터를 주고받기 위한 약속된 통로다. REST API는 웹에서 많이 쓰는 방식으로, 주소와 요청 방법을 조합해 작업을 표현한다.

요청 의미 예시
GET 조회 글 목록 가져오기
POST 생성 새 글 작성
PATCH 수정 글 내용 변경
DELETE 삭제 글 삭제

JSON과 XML

JSON과 XML은 데이터를 주고받는 형식이다. JSON은 간결해서 요즘 웹 API에서 자주 쓰이고, XML은 공공기관 API나 오래된 시스템에서 많이 볼 수 있다.

공공데이터 API 문서를 읽을 때 XML 응답, serviceKey, 지역코드, 계약월 같은 필수 파라미터가 나오는 이유도 이 흐름과 연결된다.

데이터베이스와 CRUD

데이터베이스는 서비스의 기억 공간이다. 회원, 주문, 게시글, 결제 내역 같은 데이터를 저장한다. 기본 동작은 CRUD로 정리한다.

Create는 만들기, Read는 읽기, Update는 수정하기, Delete는 삭제하기다. SQL에서는 각각 INSERT, SELECT, UPDATE, DELETE와 연결된다.

오늘 가져갈 것

웹서비스의 핵심 흐름은 클라이언트 요청, 서버 처리, 데이터베이스 조회, 응답이다. API와 DB를 함께 이해해야 화면 뒤에서 무슨 일이 일어나는지 설명할 수 있다.
