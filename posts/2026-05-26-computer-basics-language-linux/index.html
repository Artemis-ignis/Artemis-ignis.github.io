2026-05-26 이어드림스쿨 수업 정리: 컴퓨터는 코드를 어떻게 이해할까

오늘 컴퓨터의 이해 자료에서 가장 크게 잡은 감각은 이것이다. 사람은 사람이 읽기 쉬운 말로 코드를 쓰고, 컴퓨터는 그 코드를 바로 읽는 게 아니라 자기 방식으로 번역해서 실행한다.

처음에는 “파이썬 코드를 치면 컴퓨터가 그대로 알아듣는 것 아닌가?“라고 생각하기 쉽다. 그런데 실제로는 중간에 번역자가 있다. 그 번역자가 인터프리터이거나 컴파일러다.

코드가 실행되는 흐름

흐름만 보면 생각보다 단순하다.

  1. 사람이 코드를 작성한다.
  2. 인터프리터나 컴파일러가 코드를 번역한다.
  3. 컴퓨터가 이해할 수 있는 형태로 바뀐다.
  4. 컴퓨터가 명령을 실행한다.

인터프리터는 코드를 한 줄씩 읽고 실행하는 방식이다. 파이썬이 대표적이다. 선생님이 문장을 한 줄씩 읽어주면 학생이 바로 따라 하는 느낌에 가깝다.

컴파일러는 실행 전에 코드를 한 번에 번역하는 방식이다. C나 C++이 대표적이다. 책 전체를 먼저 번역해두고, 그 번역본을 실행하는 느낌이다.

사람에게 가까운 언어와 컴퓨터에 가까운 언어

프로그래밍 언어는 크게 보면 사람에게 가까운 언어와 컴퓨터에 가까운 언어로 나눌 수 있다.

구분 느낌 예시
High Level Language 사람이 읽기 쉬움 Python, JavaScript, R
Low Level Language 컴퓨터에 가까움 기계어, 어셈블리어, C의 일부 영역

파이썬, 자바스크립트, R은 사람이 읽기 쉬운 편이다. 그래서 처음 배울 때 접근성이 좋고, 빠르게 결과물을 만들 수 있다.

반대로 기계어, 어셈블리어, C의 일부 영역은 컴퓨터에 더 가깝다. 배우기는 어렵지만 속도나 메모리 같은 부분을 세밀하게 다룰 수 있다.

이 구분이 중요한 이유는 “왜 어떤 언어는 배우기 쉽고, 어떤 언어는 빠르다고 하는지"를 이해하게 해주기 때문이다. 데이터 분석을 빠르게 시작하려면 파이썬이 좋고, 시스템을 아주 세밀하게 다뤄야 하면 더 낮은 수준의 언어가 필요할 수 있다.

리눅스를 배우는 이유

수업에서 리눅스가 자주 언급되는 이유도 여기서 이어진다. 웹사이트나 서비스를 실제로 배포하고 운영할 때 서버 환경이 리눅스인 경우가 많다.

리눅스를 많이 쓰는 이유는 대략 이렇다.

  1. 실제 서버 환경에서 많이 사용된다.
  2. 비용이 낮고 운영 자동화에 좋다.
  3. 24시간 켜두는 서버 운영에 안정적이다.
  4. 배포, 로그 확인, 자동화 도구가 리눅스 중심인 경우가 많다.

윈도우에서 개발하더라도 WSL Ubuntu 같은 환경을 익혀두면 좋다. 내 컴퓨터에서 코드를 쓰는 것과 실제 서버에 올려 운영하는 것은 다른 일이기 때문이다.

내 식으로 정리

컴퓨터를 배운다는 건 부품 이름을 외우는 것에서 끝나지 않는다. 내가 쓴 코드가 어떤 과정을 거쳐 실행되는지, 그리고 실제 서비스가 어떤 환경에서 돌아가는지 이해하는 일이다.
