2026-05-26 이어드림스쿨 수업 정리: 앱·웹 서비스는 화면 뒤에서 어떻게 돌아갈까

앱_웹 서비스를 위한 모든 것 자료에서 핵심은 하나였다. 서비스는 예쁜 화면만으로 완성되지 않는다. 사용자가 보는 화면 뒤에는 서버, 데이터베이스, 관리자 기능, 보안, 개인정보 처리까지 줄줄이 붙어 있다.

초보자는 화면을 먼저 본다. 버튼이 있고, 입력창이 있고, 결과가 보이면 서비스처럼 느껴진다. 하지만 실제 운영 가능한 서비스는 버튼을 누른 뒤에 무슨 일이 일어나는지가 더 중요하다.

클라이언트와 서버

클라이언트는 사용자가 직접 보는 쪽이다. 웹브라우저, 스마트폰 앱, 화면에 보이는 버튼과 입력창이 여기에 가깝다.

서버는 뒤에서 실제 일을 처리하는 컴퓨터다. 사용자가 요청한 내용을 확인하고, 데이터베이스에서 자료를 찾고, 필요한 계산이나 검증을 한 뒤 응답을 돌려준다.

식당으로 비유하면 클라이언트는 손님이 보는 주문 화면이고, 서버는 주방과 계산대 뒤쪽 처리 공간이다. 손님이 주문하면 주방은 재료가 있는지 확인하고 음식을 만든 뒤 결과를 돌려준다.

요청과 응답

웹서비스는 요청과 응답의 반복이다.

사용자 행동 → 클라이언트 요청 → 서버 처리 → 데이터베이스 조회 또는 저장 → 서버 응답 → 화면 표시

예를 들어 사용자가 게시판에서 글 목록 버튼을 누르면 클라이언트는 서버에 “글 목록을 주세요"라고 요청한다. 서버는 데이터베이스에서 글을 찾아 응답하고, 클라이언트는 그 결과를 화면에 보여준다.

여기서 중요한 점은 사용자는 버튼 하나만 봐도, 뒤에서는 여러 단계가 움직인다는 것이다.

백엔드와 비즈니스 로직

백엔드는 사용자가 잘 보지 못하는 뒤쪽 영역이다. 로그인 확인, 데이터 저장, 추천 계산, 결제 처리, DB 조회 같은 일을 담당한다.

비즈니스 로직은 서비스에서 정한 규칙이다. 예를 들면 이런 것들이다.

규칙 백엔드에서 하는 판단
재고가 없으면 구매 불가 상품 수량을 확인한다.
회원만 댓글 작성 가능 로그인 상태를 확인한다.
쿠폰은 한 번만 사용 가능 사용 이력을 확인한다.
5만 원 이상이면 무료배송 주문 금액을 계산한다.

화면은 입구이고, 백엔드는 규칙을 지키게 하는 담당자다. 그래서 서비스를 만들 때는 화면 설계와 백엔드 설계를 같이 생각해야 한다.

명함 촬영 서비스로 본 기획

명함 촬영 서비스를 예로 들면 일반 사용자는 명함을 촬영하고 사진 정보를 전송한다. 여기까지만 보면 간단해 보인다.

하지만 실제 서비스로 생각하면 할 일이 늘어난다.

  1. 사용자는 명함 이미지를 업로드한다.
  2. 서버는 이미지를 저장하거나 분석한다.
  3. 관리자는 자동 추출 결과를 검수한다.
  4. 잘못 인식된 정보는 수정된다.
  5. 사용자는 삭제 요청을 할 수 있어야 한다.

명함에는 이름, 회사, 전화번호, 이메일 같은 개인정보가 들어갈 수 있다. 그래서 수집 동의, 접근 권한, 삭제 요청, 보관 기준을 같이 생각해야 한다.

초보자는 사용자 화면만 본다. 조금 더 깊게 보면 관리자 페이지가 보인다. 더 깊게 보면 개인정보와 법령, 보안, 운영 정책이 보인다.

내 식으로 정리

웹서비스를 만든다는 건 화면을 그리는 일이면서 동시에 흐름을 설계하는 일이다. 누가 요청하고, 서버가 무엇을 확인하고, 어떤 데이터를 저장하며, 관리자는 어디서 검수하고, 개인정보는 어떻게 보호할지까지 연결해서 봐야 한다.
