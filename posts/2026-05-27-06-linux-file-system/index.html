2026-05-27 강의자료 06: 리눅스 파일 시스템

이 글은 [강의자료] 리눅스 파일 시스템.pdf를 보고 다시 정리한 복습 기록이다. 슬라이드를 그대로 옮기기보다는, 나중에 봤을 때 “아, 이게 그 말이었지” 하고 떠올릴 수 있게 쉬운 말과 예시 중심으로 적었다.

한 줄 요약

리눅스 파일 시스템은 /를 꼭대기로 하는 하나의 큰 나무 구조이며, 각 디렉터리는 대체로 맡은 역할이 정해져 있다.

수업 흐름

Windows는 드라이브 문자로 시작하는 경로가 익숙하지만, 리눅스는 /라는 루트에서 모든 경로가 시작된다. /home에는 사용자 파일이, /etc에는 설정 파일이, /var에는 로그처럼 자주 변하는 파일이, /bin이나 /usr/bin에는 실행 파일이 들어간다. 위치의 의미를 알면 서버에서 파일을 찾을 때 덜 헤맨다.

쉽게 이해하기

리눅스 디렉터리는 학교 건물 배치도처럼 볼 수 있다. 교무실, 교실, 창고, 게시판 위치가 정해져 있으면 처음 온 사람도 안내판을 보고 찾을 수 있다. /etc, /var, /home 같은 이름은 그 안내판 역할을 한다.

헷갈렸던 단어

용어 내가 이해한 뜻
/ 리눅스 파일 시스템의 최상위 루트
/home 일반 사용자의 개인 파일이 주로 들어가는 위치
/etc 시스템과 프로그램 설정 파일이 모이는 위치
/var 로그, 캐시처럼 실행 중 계속 변하는 파일이 많은 위치

짧은 예시

서버 오류를 확인할 때 /var/log 아래의 로그 파일을 보는 경우가 많다. 설정을 바꿀 때는 /etc 아래 파일을 수정할 수 있다. 이때 권한이 필요할 수 있으므로 아무 파일이나 함부로 바꾸면 안 된다.

복습 체크

  • 리눅스 경로가 /에서 시작한다는 점을 이해했는가?
  • /home, /etc, /var의 대략적인 역할을 말할 수 있는가?
  • 설정 파일 수정 전 백업이나 변경 기록이 필요하다는 점을 떠올릴 수 있는가?

오늘의 정리

리눅스 파일 시스템은 처음에는 낯설지만, 자주 가는 위치부터 익히면 길 찾기처럼 익숙해진다. 서버에서 문제를 해결할 때 파일 위치를 아는 것만으로도 절반은 편해진다.
