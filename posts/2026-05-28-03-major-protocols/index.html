2026-05-28 강의자료 03: 주요 프로토콜

프로토콜은 통신 약속이다. 컴퓨터끼리 데이터를 주고받으려면 서로 같은 규칙을 이해해야 한다.

주요 프로토콜 정리

프로토콜 역할
IP 목적지 주소를 기준으로 데이터 전달
DNS 도메인 이름을 IP 주소로 변환
DHCP 네트워크 장치에 IP 자동 할당
TCP 신뢰성 있는 연결 중심 통신
UDP 빠른 전송이 중요한 통신
FTP 파일 전송
SMTP 이메일 전송
SSH 원격 서버 접속
SSL/TLS 암호화 통신

웹서비스와 연결하기

브라우저에 도메인을 입력하면 DNS가 IP를 찾고, HTTP 요청이 서버로 이동한다. 로그인이나 결제처럼 민감한 데이터가 오갈 때는 SSL/TLS로 암호화된 HTTPS를 사용한다.

서버에 직접 접속해 배포나 로그 확인을 할 때는 SSH를 많이 사용한다.

오늘 가져갈 것

프로토콜을 알면 웹서비스가 단순히 “인터넷으로 연결된다"가 아니라 어떤 약속들이 이어져 동작하는지 보인다. 문제를 만났을 때 DNS인지, 포트인지, 인증서인지 나눠 생각할 수 있다.
