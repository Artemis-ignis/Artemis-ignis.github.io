2026-05-29 강의자료 03: 리스트 - 모아모아 다모아

리스트는 여러 값을 하나로 묶어 저장하는 자료형이다. 학생 이름, 점수 목록, 상품 목록처럼 비슷한 성격의 값을 함께 다룰 때 유용하다. 리스트 만들기 scores = [90, 80, 70] names = ["사용자 A", "사용자 B", "사용자 C"] 리스트는 대괄호를 사용하고, 값들은 쉼표로 구분한다. 인덱스 리스트의 위치 번호를 인덱스라고 한다. 파이썬에서 첫 번째 위치는 1이 아니라 0이다. scores = [90, 80, 70] print(scores[0]) print(scores[1]) scores[0]은 첫 번째 값인 90을 의미한다. 반복과 활용 리스트는 반복문과 함께 쓰면 더 강력하다. names = ["사용자 A", "사용자 B", "사용자 C"] for name in names: print(name) 오늘 가져갈 것 리스트는 데이터를 여러 개 다루기 위한 기본 도구다. 데이터 분석에서도 표의 열이나 행을 다룰 때 여러 값을 묶어 처리하는 사고가 필요하다.

2026-05-29 강의자료 02: 조건문 - 진실 혹은 거짓

조건문은 상황에 따라 다른 코드를 실행하게 해준다. 컴퓨터에게 “이 조건이면 이렇게 하고, 아니면 저렇게 해"라고 알려주는 문법이다. 입력받기 input()은 사용자에게 값을 입력받는다. 입력받은 값은 기본적으로 문자열이기 때문에 숫자 계산을 하려면 자료형을 바꿔야 한다. age = int(input("나이를 입력하세요: ")) if 조건문 조건문은 비교 결과가 참인지 거짓인지에 따라 실행 흐름을 나눈다. score = 85 if score >= 80: print("통과") else: print("다시 도전") 파이썬에서 들여쓰기는 문법이다. 들여쓰기가 틀리면 코드의 의미가 달라지거나 오류가 난다. 비교 연산 ==, !=, >, <, >=, <= 같은 비교 연산자는 조건을 만들 때 사용한다.

2026-05-29 강의자료 01: 기초 자료형 - Python으로의 초대

이 자료는 파이썬 문법의 출발점이다. 코드를 처음 배울 때 가장 먼저 익혀야 할 것은 값을 출력하고, 변수에 저장하고, 자료형을 구분하는 일이다. 출력과 변수 print()는 값을 화면에 보여주는 함수다. 변수는 값을 담는 이름표다. name = "사용자 A" age = 20 print(name) print(age) 변수 이름은 의미 있게 짓는 것이 좋다. x보다 user_name, total_price처럼 읽히는 이름이 나중에 코드를 이해하기 쉽다. 자료형 자료형은 값의 종류다. 정수, 실수, 문자열, 참/거짓 같은 값은 서로 다른 방식으로 다뤄진다. 자료형 예시 int 10 float 3.14 str "hello" bool True 숫자 10과 문자열 "10"은 다르다. 계산을 하려면 숫자 자료형이어야 한다.

2026-05-28 강의자료 04: 보안

보안은 서비스를 만든 뒤 추가로 붙이는 장식이 아니다. 로그인, 개인정보, 권한, 데이터 저장을 다루는 순간부터 함께 생각해야 한다. 해시 해시는 데이터를 일정한 규칙으로 바꾼 값이다. 비밀번호는 원문 그대로 저장하면 위험하기 때문에 보통 해시를 사용한다. 중요한 점은 원래 값을 쉽게 되돌릴 수 없게 관리해야 한다는 것이다. 쿠키와 세션 HTTP는 기본적으로 이전 요청을 기억하지 않는다. 로그인 상태를 유지하려면 쿠키나 세션 같은 방법이 필요하다. 쿠키는 브라우저에 저장되는 작은 정보이고, 세션은 서버가 사용자 상태를 기억하는 방식이다.

2026-05-28 강의자료 03: 주요 프로토콜

프로토콜은 통신 약속이다. 컴퓨터끼리 데이터를 주고받으려면 서로 같은 규칙을 이해해야 한다. 주요 프로토콜 정리 프로토콜 역할 IP 목적지 주소를 기준으로 데이터 전달 DNS 도메인 이름을 IP 주소로 변환 DHCP 네트워크 장치에 IP 자동 할당 TCP 신뢰성 있는 연결 중심 통신 UDP 빠른 전송이 중요한 통신 FTP 파일 전송 SMTP 이메일 전송 SSH 원격 서버 접속 SSL/TLS 암호화 통신 웹서비스와 연결하기 브라우저에 도메인을 입력하면 DNS가 IP를 찾고, HTTP 요청이 서버로 이동한다. 로그인이나 결제처럼 민감한 데이터가 오갈 때는 SSL/TLS로 암호화된 HTTPS를 사용한다.
