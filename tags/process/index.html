2026-05-27 강의자료 08: 프로세스와 네트워크

서버 운영에서는 프로세스와 네트워크를 함께 봐야 한다. 웹 애플리케이션이 실행 중이어도 포트가 열려 있지 않으면 외부에서 접속할 수 없고, 네트워크는 열려 있어도 프로세스가 죽으면 응답하지 않는다. 프로세스 확인 서버에서 서비스가 동작하는지 확인하려면 먼저 프로세스를 본다. 어떤 프로그램이 실행 중인지, CPU나 메모리를 얼마나 쓰는지 확인한다. 리눅스에서는 ps, top, htop 같은 도구를 사용할 수 있다. 문제가 생기면 해당 프로세스를 종료하거나 재시작해야 할 때도 있다. 네트워크 확인 웹서비스는 특정 포트에서 요청을 기다린다. 예를 들어 개발 서버는 3000, 5173, 8000, 1313 같은 포트를 사용할 수 있다.

2026-05-27 강의자료 02: 프로세스

프로세스는 실행 중인 프로그램이다. 프로그램은 저장 장치에 있는 파일이고, 그 프로그램이 실제로 실행되어 메모리에 올라오면 프로세스가 된다. 프로그램과 프로세스 메모장 프로그램 파일이 있다고 해서 항상 실행 중인 것은 아니다. 사용자가 메모장을 열면 운영체제가 실행 공간을 만들고, 그때부터 프로세스가 된다. 프로세스는 코드, 데이터, 실행 상태, 메모리 공간을 가진다. 운영체제는 여러 프로세스가 동시에 움직일 수 있도록 스케줄링한다. 왜 중요한가 서버에서 웹 애플리케이션을 실행하면 그것도 하나의 프로세스다. 서비스가 멈췄을 때 프로세스가 살아 있는지, CPU나 메모리를 너무 많이 쓰고 있는지 확인해야 한다.
