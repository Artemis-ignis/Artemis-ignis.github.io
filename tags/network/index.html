2026-05-28 강의자료 01: 네트워크 개요

이 글은 [수업 자료] 네트워크 개요.pdf를 보고 다시 정리한 복습 기록이다. 슬라이드를 그대로 옮기기보다는, 나중에 봤을 때 “아, 이게 그 말이었지” 하고 떠올릴 수 있게 쉬운 말과 예시 중심으로 적었다. 한 줄 요약 네트워크는 여러 컴퓨터가 약속된 주소와 규칙을 이용해 데이터를 작은 조각으로 주고받는 구조다. 수업 흐름 인터넷을 쓸 때 우리는 주소창에 도메인을 입력하지만, 실제 통신은 IP 주소와 패킷을 중심으로 움직인다. DNS가 도메인을 IP로 바꿔 주고, 데이터는 작은 패킷으로 나뉘어 여러 장비를 지나 목적지까지 간다. 라우터와 스위치 같은 장비는 데이터를 어디로 보낼지 결정한다.

2026-05-27 강의자료 08: 프로세스와 네트워크

이 글은 [강의자료] 프로세스와 네트워크.pdf를 보고 다시 정리한 복습 기록이다. 슬라이드를 그대로 옮기기보다는, 나중에 봤을 때 “아, 이게 그 말이었지” 하고 떠올릴 수 있게 쉬운 말과 예시 중심으로 적었다. 한 줄 요약 서버 문제를 볼 때는 어떤 프로세스가 떠 있는지, 어떤 포트를 쓰는지, 네트워크 요청이 어디서 막히는지 확인해야 한다. 수업 흐름 리눅스 서버에서는 프로그램이 프로세스로 실행되고, 외부 요청은 네트워크 포트를 통해 들어온다. 웹서버가 정상 실행 중인지 보려면 프로세스 목록을 보고, 포트가 열려 있는지 확인하고, 필요하면 로그를 본다. 프로세스와 네트워크를 따로 보지 않고 같이 봐야 실제 서비스 상태를 이해할 수 있다.
