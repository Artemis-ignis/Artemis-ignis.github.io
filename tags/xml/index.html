2026-05-26 강의자료 03: 데이터베이스와 REST API

이 자료는 웹서비스가 데이터를 어떻게 주고받고 저장하는지 설명한다. 화면에서 버튼을 누르는 일은 단순해 보이지만, 뒤에서는 API 요청과 데이터베이스 처리가 이어진다. API와 REST API API는 프로그램끼리 데이터를 주고받기 위한 약속된 통로다. REST API는 웹에서 많이 쓰는 방식으로, 주소와 요청 방법을 조합해 작업을 표현한다. 요청 의미 예시 GET 조회 글 목록 가져오기 POST 생성 새 글 작성 PATCH 수정 글 내용 변경 DELETE 삭제 글 삭제 JSON과 XML JSON과 XML은 데이터를 주고받는 형식이다. JSON은 간결해서 요즘 웹 API에서 자주 쓰이고, XML은 공공기관 API나 오래된 시스템에서 많이 볼 수 있다.

2026-05-26 이어드림스쿨 수업 정리: REST API와 데이터베이스 기초

데이터베이스와 REST API 자료는 오늘 내용 중 가장 서비스 개발에 가까운 파트였다. 웹서비스가 화면과 서버의 대화라면, API는 그 대화의 약속이고 데이터베이스는 기억을 저장하는 공간이다. 처음엔 용어가 많아서 복잡해 보이지만, 큰 흐름은 이렇다. 클라이언트가 API로 요청한다 → 서버가 비즈니스 로직을 처리한다 → 서버가 DB에 쿼리한다 → 결과를 응답한다 API와 REST API API는 프로그램끼리 데이터를 주고받는 약속된 통로다. 사람이 메뉴판을 보고 주문하듯이, 프로그램도 API라는 약속을 보고 요청한다. REST API는 웹에서 많이 쓰는 API 방식이다. 주소와 요청 방식으로 데이터를 다룬다.
