2026-05-27 강의자료 04: 파일 시스템

파일 시스템은 컴퓨터가 파일과 디렉터리를 정리하는 방식이다. 사용자는 폴더와 파일을 눈으로 보지만, 운영체제는 정해진 규칙에 따라 저장 위치와 이름을 관리한다. 파일과 디렉터리 파일은 데이터를 담는 단위이고, 디렉터리는 파일을 묶어두는 공간이다. 디렉터리 안에 또 다른 디렉터리가 들어갈 수 있어서 트리 구조가 만들어진다. 경로는 파일의 위치를 나타낸다. 윈도우에서는 C:\Users\... 같은 형태를 많이 쓰고, 리눅스에서는 /home/user/... 같은 형태를 많이 쓴다. 개발에서 중요한 이유 개발할 때는 파일 위치가 자주 문제를 만든다. 실행 위치가 다르면 상대 경로가 달라지고, 파일 권한이 없으면 읽거나 쓸 수 없다.

2026-05-27 강의자료 03: 메모리

메모리는 프로그램이 실행되는 동안 데이터를 올려두는 작업 공간이다. 저장 장치가 책장이라면, 메모리는 지금 펼쳐 놓고 쓰는 책상에 가깝다. 메모리가 하는 일 프로그램을 실행하면 운영체제는 필요한 코드와 데이터를 메모리에 올린다. CPU는 메모리에 올라온 데이터를 빠르게 읽고 계산한다. 메모리가 부족하면 컴퓨터는 느려진다. 책상 위 공간이 부족하면 책을 계속 꺼냈다 넣었다 해야 하는 것처럼, 컴퓨터도 데이터를 옮기는 데 시간이 더 든다. 개발과 메모리 개발자는 변수를 만들고 리스트에 값을 넣고 객체를 생성한다. 이런 값들은 실행 중에 메모리를 사용한다. 데이터가 많아지는 프로그램에서는 메모리 사용량을 생각해야 한다.

2026-05-27 강의자료 02: 프로세스

프로세스는 실행 중인 프로그램이다. 프로그램은 저장 장치에 있는 파일이고, 그 프로그램이 실제로 실행되어 메모리에 올라오면 프로세스가 된다. 프로그램과 프로세스 메모장 프로그램 파일이 있다고 해서 항상 실행 중인 것은 아니다. 사용자가 메모장을 열면 운영체제가 실행 공간을 만들고, 그때부터 프로세스가 된다. 프로세스는 코드, 데이터, 실행 상태, 메모리 공간을 가진다. 운영체제는 여러 프로세스가 동시에 움직일 수 있도록 스케줄링한다. 왜 중요한가 서버에서 웹 애플리케이션을 실행하면 그것도 하나의 프로세스다. 서비스가 멈췄을 때 프로세스가 살아 있는지, CPU나 메모리를 너무 많이 쓰고 있는지 확인해야 한다.

2026-05-27 강의자료 01: 운영체제 소개

운영체제는 컴퓨터를 사용할 수 있게 해주는 기본 소프트웨어다. 사용자는 프로그램을 실행한다고 느끼지만, 실제로는 운영체제가 CPU, 메모리, 파일, 입출력 장치를 관리한다. 운영체제의 역할 운영체제는 컴퓨터의 관리자다. 여러 프로그램이 동시에 실행될 때 어떤 프로그램에 CPU 시간을 줄지, 메모리를 얼마나 배정할지, 파일을 어디에 저장할지 조율한다. 예를 들어 브라우저, 메신저, 코드 편집기를 동시에 켜도 컴퓨터가 어느 정도 질서 있게 움직이는 이유는 운영체제가 자원을 나눠주기 때문이다. 사용자와 하드웨어 사이 사용자는 하드웨어를 직접 제어하지 않는다. 파일을 저장하거나 프로그램을 실행하는 행동은 운영체제를 통해 하드웨어에 전달된다.
