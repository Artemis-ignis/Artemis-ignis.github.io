2026-05-27 강의자료 08: 프로세스와 네트워크

서버 운영에서는 프로세스와 네트워크를 함께 봐야 한다. 웹 애플리케이션이 실행 중이어도 포트가 열려 있지 않으면 외부에서 접속할 수 없고, 네트워크는 열려 있어도 프로세스가 죽으면 응답하지 않는다. 프로세스 확인 서버에서 서비스가 동작하는지 확인하려면 먼저 프로세스를 본다. 어떤 프로그램이 실행 중인지, CPU나 메모리를 얼마나 쓰는지 확인한다. 리눅스에서는 ps, top, htop 같은 도구를 사용할 수 있다. 문제가 생기면 해당 프로세스를 종료하거나 재시작해야 할 때도 있다. 네트워크 확인 웹서비스는 특정 포트에서 요청을 기다린다. 예를 들어 개발 서버는 3000, 5173, 8000, 1313 같은 포트를 사용할 수 있다.

2026-05-27 강의자료 06: 리눅스 파일 시스템

리눅스 파일 시스템은 /를 기준으로 시작한다. 윈도우의 C 드라이브처럼 여러 드라이브가 먼저 보이는 구조와 다르게, 리눅스는 하나의 루트 아래에 파일과 디렉터리가 정리된다. 주요 디렉터리 경로 의미 /home 사용자 개인 파일 /etc 설정 파일 /var 로그와 변하는 데이터 /bin 기본 실행 파일 /usr 사용자 프로그램과 라이브러리 /tmp 임시 파일 서버에서 설정을 바꿀 때 /etc, 로그를 볼 때 /var/log, 사용자 작업 파일을 찾을 때 /home을 자주 만난다. 절대 경로와 상대 경로 절대 경로는 루트부터 시작하는 전체 주소다. 상대 경로는 현재 위치를 기준으로 한 주소다.

2026-05-27 강의자료 05: 리눅스의 세계로

리눅스는 서버 환경에서 많이 쓰이는 운영체제다. 개인 컴퓨터에서는 윈도우를 많이 쓰지만, 웹서비스를 배포하고 운영하는 서버에서는 리눅스를 자주 만난다. 리눅스를 배우는 이유 리눅스는 안정적이고 비용 부담이 낮으며, 서버 자동화 도구와 잘 연결된다. 클라우드 서버를 만들면 기본 환경이 리눅스인 경우도 많다. 개발자가 로그를 확인하거나, 파일을 수정하거나, 서버 프로세스를 재시작할 때 리눅스 명령어를 사용한다. CLI 환경 CLI는 글자로 명령을 입력하는 환경이다. 처음에는 버튼이 없어서 어렵게 느껴지지만, 반복 작업을 자동화하기 좋다. 예를 들어 ls로 파일 목록을 보고, cd로 디렉터리를 이동하고, mkdir로 폴더를 만든다. 이런 명령어가 익숙해지면 서버에서 직접 문제를 확인할 수 있다.

2026-05-26 이어드림스쿨 수업 정리: 컴퓨터는 코드를 어떻게 이해할까

오늘 컴퓨터의 이해 자료에서 가장 크게 잡은 감각은 이것이다. 사람은 사람이 읽기 쉬운 말로 코드를 쓰고, 컴퓨터는 그 코드를 바로 읽는 게 아니라 자기 방식으로 번역해서 실행한다. 처음에는 “파이썬 코드를 치면 컴퓨터가 그대로 알아듣는 것 아닌가?“라고 생각하기 쉽다. 그런데 실제로는 중간에 번역자가 있다. 그 번역자가 인터프리터이거나 컴파일러다. 코드가 실행되는 흐름 흐름만 보면 생각보다 단순하다. 사람이 코드를 작성한다. 인터프리터나 컴파일러가 코드를 번역한다. 컴퓨터가 이해할 수 있는 형태로 바뀐다. 컴퓨터가 명령을 실행한다. 인터프리터는 코드를 한 줄씩 읽고 실행하는 방식이다. 파이썬이 대표적이다. 선생님이 문장을 한 줄씩 읽어주면 학생이 바로 따라 하는 느낌에 가깝다.
