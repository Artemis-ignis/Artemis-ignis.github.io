2026-05-27 강의자료 06: 리눅스 파일 시스템

이 글은 [강의자료] 리눅스 파일 시스템.pdf를 보고 다시 정리한 복습 기록이다. 슬라이드를 그대로 옮기기보다는, 나중에 봤을 때 “아, 이게 그 말이었지” 하고 떠올릴 수 있게 쉬운 말과 예시 중심으로 적었다. 한 줄 요약 리눅스 파일 시스템은 /를 꼭대기로 하는 하나의 큰 나무 구조이며, 각 디렉터리는 대체로 맡은 역할이 정해져 있다. 수업 흐름 Windows는 드라이브 문자로 시작하는 경로가 익숙하지만, 리눅스는 /라는 루트에서 모든 경로가 시작된다. /home에는 사용자 파일이, /etc에는 설정 파일이, /var에는 로그처럼 자주 변하는 파일이, /bin이나 /usr/bin에는 실행 파일이 들어간다. 위치의 의미를 알면 서버에서 파일을 찾을 때 덜 헤맨다.

2026-05-27 강의자료 04: 파일 시스템

이 글은 [수업자료] 파일 시스템.pdf를 보고 다시 정리한 복습 기록이다. 슬라이드를 그대로 옮기기보다는, 나중에 봤을 때 “아, 이게 그 말이었지” 하고 떠올릴 수 있게 쉬운 말과 예시 중심으로 적었다. 한 줄 요약 파일 시스템은 데이터를 파일과 폴더 구조로 정리하고, 어디에 무엇이 있는지 운영체제가 찾을 수 있게 만드는 방식이다. 수업 흐름 우리가 보는 파일 이름 뒤에는 경로, 확장자, 크기, 생성 시간, 수정 시간, 권한 같은 정보가 붙어 있다. 운영체제는 이 정보를 이용해 파일을 찾고 읽고 쓴다. 파일 시스템이 없다면 저장장치 안의 데이터는 그냥 긴 숫자 덩어리처럼 보일 것이다.
